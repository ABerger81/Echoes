# Devlog v0.7.0

Date: 2026-06-23

## Goal

Heartbeat & Tension System ‚Äî add the state machine that drives all tension, audio, and visual feedback across the rest of the game. Every future milestone (audio, safe zones, monster AI) hooks into this.

---

## Completed

**Milestone 7 ‚Äî Heartbeat & Tension System:**

- HeartbeatManager.cs ‚Äî four-state machine (Calm / Alert / Fear / Panic) driven by a hidden `_noisePressure` float (0‚Äì1). Noise decays over time; player actions raise it. State transitions fire `OnStateChanged` for all subscribers.
- BreathingLevel ‚Äî separate continuous float (0‚Äì1) that follows HeartbeatState with inertia: rises fast (rate 3.0), decays slowly (rate 0.1). Monster system (M12) will read this as a detectable noise source even while the player is stationary.
- PlayerNoiseEmitter.cs ‚Äî reads `StarterAssetsInputs.sprint` directly to distinguish walk from sprint. Reports continuous noise each frame while moving. Subscribes to `Treasure.OnCollected` to fire a one-time burst on Minor Treasure pickup.
- HeartbeatVisuals.cs ‚Äî placeholder vignette overlay using a CanvasGroup on a fullscreen black Image. Alpha set per state: Alert 0.1, Fear 0.25, Panic 0.45. To be replaced with URP Volume effect in M8.
- GameManager integration ‚Äî `OnEscapeTriggered` forces `_noisePressure = 1.0` and Panic state immediately when the Major Treasure is collected.
- Static field reset on `OnDestroy()` ‚Äî same pattern as GameManager; prevents stale `CurrentState` and `BreathingLevel` on the next Play session in the editor.

## Problems Encountered

- **Gravity noise bug** ‚Äî `CharacterController.velocity.magnitude` includes Y (~2 m/s from gravity when grounded). Standing still was reporting Walk-level noise. Fix: use horizontal-only magnitude `new Vector3(velocity.x, 0f, velocity.z).magnitude`.
- **Sprint threshold oscillation** ‚Äî initial implementation used a `sprintThreshold` float to infer sprint from speed. With `MoveSpeed = 4` and `sprintThreshold = 4`, the player oscillated between Walk and Sprint hundreds of times per second. Fix: replaced the threshold entirely with a direct `StarterAssetsInputs.sprint` read.
- **Compiler warning CS0414** ‚Äî `_lastNoiseCategory` debug field was left assigned but unread after the Update() refactor. Removed during cleanup.

---

## Lessons Learned

- `CharacterController.velocity` always includes Y from gravity ‚Äî always strip the Y axis when checking movement speed.
- Reading the input bool directly (`StarterAssetsInputs.sprint`) is more robust than inferring intent from speed. Speed thresholds are fragile: any Inspector change to MoveSpeed or SprintSpeed breaks them silently.
- Static fields survive stopping Play in the Unity editor. Reset them in `OnDestroy()` to ensure a clean start each run.
- Inspector values changed during Play mode are discarded when Play stops. Changes that need to persist must be made in Edit mode and saved with Ctrl+S before pressing Play.
