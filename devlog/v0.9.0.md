# Devlog v0.9.0

Date: 2026-06-25

## Goal

Safe Zones — binary protection zones where movement noise is suppressed and the player must wait for their elevated breathing to calm before safely moving again. The two-stage hiding experience (position hidden first, breathing must calm second) is the payoff for the heartbeat/breathing system built in M7–M8.

---

## Completed

**Milestone 9 — Safe Zones:**

- `SafeZone.cs` — new script in `Assets/_Game/Scripts/Environment/`. Trigger volume detects player entry and exit via `OnTriggerEnter` / `OnTriggerExit`. Calls `HeartbeatManager.EnterSafeZone()` and `ExitSafeZone()`.
- HeartbeatManager extended — `IsInSafeZone` static bool (read by monster system in M12); `EnterSafeZone()` and `ExitSafeZone()` instance methods; `SetContinuousNoise()` early-returns when `IsInSafeZone` is true.
- Noise suppression — movement stops raising `_noisePressure` while inside. Existing decay (`noiseDecayRate * Time.deltaTime`) continues with no extra code. State steps down naturally: Panic → Fear → Alert → Calm.
- `AddNoiseBurst` still fires inside — design intent: safe zones protect from movement noise, not from reckless treasure collection.
- Static field reset — `IsInSafeZone` reset in both `Awake()` and `OnDestroy()`, consistent with `CurrentState` and `BreathingLevel`.
- Scene placement — `SafeZone_01` placed in a dead end in Level_Minotaur.
- CaptureHandler fix — cursor unlocked on level end (`Cursor.lockState = CursorLockMode.None`); `OnApplicationFocus` re-applies unlock after alt-tab, preventing cursor re-lock on window refocus.

---

## Problems Encountered

- **EnterSafeZone set IsInSafeZone = false** — copy-paste error set the wrong value. Debug log also said "Exited" instead of "Entered". Caught on file read before testing.
- **ExitSafeZone missing entirely** — method was not written. Caught on file read.
- **Typos in SafeZone.cs method names** — `OnriggerEnter` and `OerExit` instead of `OnTriggerEnter` and `OnTriggerExit`. Unity silently ignores misnamed trigger methods; caught on file read before testing.
- **Cursor re-locks after alt-tab** — `OnApplicationFocus` fires on window regain and Unity re-applies `CursorLockMode.Locked`. Fixed by adding `OnApplicationFocus` to CaptureHandler that re-unlocks if `_levelEnded` is true.

---

## Lessons Learned

- Unity trigger methods (`OnTriggerEnter`, `OnTriggerExit`) must be named exactly — a typo compiles without error but the method is never called by the engine.
- `OnApplicationFocus(bool hasFocus)` is the correct place to handle cursor state on window focus change — not `OnEnable` or `Awake`.
- `SetContinuousNoise` using `Mathf.Max` means it can only raise pressure, never lower it. A low noise value (sneak, safe zone) does nothing if current pressure is already higher — decay happens naturally without extra logic.
