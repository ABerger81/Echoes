# Devlog v0.4.0

Date: 2026-06-20

## Goal

Basic Escape — wire the escape phase so the game has a completable loop: explore → collect Major Treasure → reach Exit → win.

---

## Completed

- GameManager extended: IsEscapeTriggered state, OnEscapeTriggered / OnLevelComplete / OnTimerExpired events, countdown timer
- ExitPoint.cs — trigger volume that detects Player + checks escape state, then fires level complete
- systems/escape_system.md — MVP implementation section added

---

## Problems Encountered

- Level complete! fires twice after Timer expired - after capture, the exit should be locked out, but IsEscapeTriggered is still true so ExitPoint keeps accepting the player.
- Multiple Level Complete! from one entry - The playerCapsule has both CapsuleCollider and a CharacterController, both tagged Player, so OnTriggerEnter fires twice per entry.

---

## Lessons Learned

- _IsLevelOver in GameManager - set true on timer expiry or level complete, whichever comes first. TriggerLevelComplete() returns early if already set. Exit after capture is now blocked.
- _triggered in ExitPoint - instance bool prevents the same exit volume from firing twice when the player's two physics components both generate an OnTriggerEvent callback.
