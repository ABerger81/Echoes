# Devlog v0.8.0

Date: 2026-06-24

## Goal

Audio Tension System — wire the Heartbeat state machine to audio. One Unity Audio Mixer Snapshot per HeartbeatState, ambience layering per state, breathing loops tied to state, and the URP Volume vignette replacing the M7 CanvasGroup placeholder.

---

## Completed

**Milestone 8 — Audio Tension System:**

- EchoesAudioMixer created — Master group with Music, Ambience, Breathing, SFX sub-groups. Ambience and Breathing each have per-layer sub-groups for independent snapshot volume control.
- Four Mixer Snapshots (Calm, Alert, Fear, Panic) — per-group volumes set for each state. Ambience layers build up as tension rises; breathing layers cross-fade per state; music group silences at Panic.
- AudioManager.cs — subscribes to `HeartbeatManager.OnStateChanged`, transitions snapshots using `AudioMixerSnapshot.TransitionTo()`. Escalation blend: 0.3s (sudden danger). De-escalation blend: 3.0s (earned calm). Exposes `PlayJumpscare(AudioClip)` API for M14.
- Ambience layering — four AudioSources (Labyrinth A/B, Cave Deep, Cave Drips) routed to Ambience sub-groups. Each layer fades in at a different state via snapshot volumes, creating progressive atmosphere without extra code.
- Breathing loops — three AudioSources (Normal/Moderate/Strong), one per non-Calm state, routed to Breathing sub-groups. Calm = silence. Mixer snapshots handle the crossfade.
- HeartbeatVisuals.cs upgraded — CanvasGroup placeholder removed. URP Volume Vignette now controlled via `postProcessVolume.profile.TryGet<Vignette>()`. Intensity lerped smoothly each frame via `Mathf.MoveTowards` rather than snapping instantly.
- All Debug.Log statements across all scripts wrapped in `#if UNITY_EDITOR` — visible during development, stripped from release builds automatically.

---

## Problems Encountered

- **Breathing AudioSources silent** — breathing AudioSources had Volume set to 0 from the original single-source plan. After switching to three sources controlled by mixer snapshots, volume needed to be 1 on each source. The mixer handles silencing, not the AudioSource.
- **Vignette snapping instantly** — `HandleStateChanged` was setting `_vignette.intensity.value` directly. Fixed by introducing `_targetIntensity` and lerping toward it in `Update()` via `Mathf.MoveTowards`.
- **Sprint reaching Panic too easily** — `sprintNoiseLevel = 0.8` pushed noise pressure above the Panic threshold (0.75) within seconds of sprinting. Lowered to 0.65: sprint alone reaches Fear; Panic requires sprint + treasure burst combined. Matches the design intent: "reckless sprinting and rapid collection is the main cause."
- **Panic decaying too fast** — `noiseDecayRate = 0.15` drained the Panic burst in ~1 second. Lowered to 0.05: Panic now lasts ~3 seconds after a burst before stepping down.

---

## Lessons Learned

- `AudioMixerSnapshot.TransitionTo()` handles audio crossfading automatically — no code needed to lerp individual AudioSource volumes when using sub-groups correctly.
- AudioSource volume and mixer group volume multiply. An AudioSource at volume 0 produces silence regardless of the mixer — the mixer cannot amplify zero.
- `#if UNITY_EDITOR` is the correct way to keep debug logs in the codebase without shipping them. Wrap once, forget forever.
- Tuning noise values (sprintNoiseLevel, noiseDecayRate) requires playtesting — initial design values are starting points, not final values.
