## Problems Encountered

- Restart button not clickable — cursor was locked and hidden during gameplay. Clicking the Restart button required a focused, visible cursor. Fix: added ReleaseCursor() to ShowWinPanel() and ShowGameOverPanel() before the panels activate.
- After restart, cursor stayed unlocked — Unity does not reset cursor state on scene reload. Fix: re-lock and hide cursor in Restart() before calling SceneManager.LoadScene().
- Cursor requires a click to lock in the editor — split-window layout means the Game view is not focused when Play is pressed. Fix: enable Maximize on Play in the Game view. Editor-only issue; builds lock the cursor immediately on launch.
- Both panels visible + "New Text" after pressing Stop — the scene was saved with both panels active and text fields at their default values. Fix: in edit mode, deactivate both panels, set ScoreText to "Score: 0", set EscapeText to empty, save scene before next Play.

---

## Lessons Learned

- Cursor lock state is not reset by SceneManager.LoadScene() — it must be set explicitly at both ends: lock in Awake (or before LoadScene), release in any method that shows a clickable UI.
- Saving the scene with panels active or text at wrong defaults causes the editor to show stale state after Stop. Always set GameObjects to their correct initial state in Edit mode before saving.
- Deleting a placeholder script (CaptureUI.cs) before the replacement (UIManager) exists means the scene briefly has a missing component reference — replace in the correct order: add UIManager and wire it, then delete CaptureUI.
