# Devlog v0.3.0

Date: 2026-06-19

## Goal

Treasure & Score System

---

## Completed

- TreasureType.cs — enum (Minor, Major) that determines pickup behavior
- Treasure.cs — implements IInteractable, fires a static event on pickup, destroys itself
- GameManager.cs — subscribes to Treasure.OnCollected, tracks score, fires OnScoreChanged for Milestone 6 UI
- systems/treasure_system.md — documents event flow and patterns used

---

## Problems Encountered

- No problems, systems/treasure_system.md was created.

---

## Lessons Learned

- Static events live on the class, not on an instance — any Treasure in the scene fires the same event without wiring
- Static events outlive GameObjects: always unsubscribe in OnDestroy() or a destroyed listener will still receive calls
- TreasureType enum over subclasses: one script + one Inspector field handles all types cleanly
