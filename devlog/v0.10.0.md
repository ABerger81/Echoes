# Devlog v0.10.0

Date: 2026-06-30

## Goal

Mythology Clue System — environmental IInteractable objects that teach the player how the monster behaves. No UI log, no hint system. The player finds, reads, and acts. Knowledge gap between clue-finder and clue-misser is the core skill dimension.

---

## Completed

**Milestone 10 — Mythology Clue System:**

- `ClueObject.cs` — new script in `Assets/_Game/Scripts/Environment/`. Implements `IInteractable`. Non-destructive: sets `_examined` on first interact, changes prompt to re-examine text, plays optional sound. Second interact does nothing.
- `_clueIntent` field — editor-only reference text per object. Never displayed at runtime. Placeholder until M12 finalises monster behaviour and confirms what each clue should claim.
- 4 symbol textures — downloaded from game-icons.net (CC BY), white-on-black PNG. Alpha Source → From Gray Scale converts white to alpha 1, black to alpha 0.
- 4 URP/Unlit Transparent materials (`Clue_Hearing`, `Clue_Return`, `Clue_SafeZone`, `Clue_Wait`) — same approach as SafeZone lyre floor marker. Decal Projectors unavailable on PC_Renderer.
- 4 `Clue_XX_TEST` GameObjects placed in Level_Minotaur — Box Collider (not trigger) + ClueObject + child Symbol Quad with matching material.

---

## Problems Encountered

None. Script compiled clean first attempt; interaction worked immediately on first test.

---

## Lessons Learned

- `GetComponent<IInteractable>()` works on interfaces — Interactor finds ClueObject via the Box Collider's GO without knowing the concrete type.
- Box Collider must NOT be set to Is Trigger — Interactor uses `Physics.Raycast`, which does not hit triggers.
